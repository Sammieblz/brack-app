# Database Table Catalog

Source of truth: remote Supabase project `waftnaqgkcgufzapcihe`, public schema, updated for club workflow changes on 2026-06-07.

All listed tables have RLS enabled. "Source" means the table is the primary write target for that domain data. "Derived" means it is computed from other source tables, triggers, RPCs, or scheduled/manual analytics work.

## Existing Risks and Duplicates

- `books` mixes canonical metadata (`title`, `author`, `isbn`, `source_provider`) with user state (`status`, `current_page`, `rating`, `notes`). This is tracked by backlog ticket 5.1/5.2.
- `book_lists.is_public` exists, but public list discovery remains out of scope for the core release.
- `posts`, `post_comments`, `post_likes`, `user_follows`, and review likes are broadly public today. This is tracked by visibility ticket 2.3.
- `profiles.current_streak`, `reading_streak_days`, and `reading_streak_history` overlap as streak-related read surfaces. `reading_streak_days` is the strongest current derived source for efficient streak computation.
- `dashboard_home_snapshots` now wraps dashboard home reads, but the refresh function still depends on `get_user_dashboard_stats` live aggregation when the snapshot is stale.
- Anonymous direct table privileges have been revoked from the public schema; unauthenticated access should go through public Edge Functions such as `search-books`.
